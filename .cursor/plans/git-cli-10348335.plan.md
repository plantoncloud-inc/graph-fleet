# Add Git CLI Support with GitHub Token Authentication

## Context

Graph-fleet currently uses Git CLI to clone repositories (e.g., project-planton) in `src/common/repos/fetcher.py`. The system will need to support private repositories in the future, requiring GitHub token authentication. Since this runs in LangGraph Cloud, the Dockerfile configuration needs to be added to `langgraph.json`.

## Changes Required

### 1. Update `langgraph.json`

Add Git CLI installation and authentication setup to the `dockerfile_lines` array:

```10:13:langgraph.json
  "dockerfile_lines": [
    "ENV HOME=/root NETRC=/root/.netrc PIP_NO_INPUT=1",
    "RUN mkdir -p /root && chmod 700 /root && printf \"machine buf.build\\nlogin %s\\npassword %s\\n\" \"$BUF_USER\" \"$BUF_API_TOKEN\" > /root/.netrc && chmod 600 /root/.netrc"
  ]
```

**New dockerfile_lines**:

- Install Git CLI package (`git`)
- Configure Git to use HTTPS with the GitHub token
- Set up `.netrc` for GitHub authentication (similar to existing buf.build pattern)
- Preserve existing buf.build authentication

The configuration will:

1. Install `git` package using `apt-get update && apt-get install -y git`
2. Configure global Git settings for credential handling
3. Add GitHub credentials to `.netrc` for seamless HTTPS cloning
4. Use `$GITHUB_TOKEN` environment variable (which LangGraph Cloud can inject)

### 2. Update Documentation

Update `README.md` to document the new `GITHUB_TOKEN` environment variable requirement for deployments that need to access private repositories.

## Environment Variable

The implementation expects a `GITHUB_TOKEN` environment variable to be configured in LangGraph Cloud deployment settings. This token should have appropriate repository access permissions for the private repositories that graph-fleet needs to clone.

## Testing Considerations

- Existing public repository cloning (project-planton if public) should continue to work
- Private repository cloning will work when `GITHUB_TOKEN` is set
- The `.netrc` file will handle authentication transparently for `git clone` commands in `fetcher.py`
